## Описание проекта

Исследовательский проект по разработке и сравнению алгоритмов автономной навигации
необитаемого подводного аппарата (НПА) в горизонтальной плоскости с использованием:

- **Классического расширенного фильтра Калмана (EKF)**
- **Адаптивного EKF** с подстройкой матрицы шума измерений R
  на основе анализа инновационной последовательности (innovation-based adaptation)

### Модель движения

Упрощённая 3-DOF кинематическая модель (2D + курс):

```
x_{k+1} = x_k + v_k * cos(ψ_k) * dt
y_{k+1} = y_k + v_k * sin(ψ_k) * dt
ψ_{k+1} = ψ_k + ω_k * dt
v_{k+1} = v_k + a_k * dt
```

Вектор состояния: `[x, y, v, ψ]`

### Параметры НПА (раздел 2.1)

| Параметр | Значение | Описание |
|----------|----------|----------|
| Масса | 50 кг | Масса аппарата |
| V_max | 2.0 м/с | Максимальная скорость |
| τ | 3.0 с | Характерное время разгона |
| ψ̇_max | 5°/с | Максимальная угловая скорость |
| Δt | 0.1 с | Шаг дискретизации |
| ΔV_max | (V_max/τ)·Δt ≈ 0.067 м/с | Макс. изменение скорости за шаг |
| Δψ_max | ψ̇_max·Δt ≈ 0.0087 рад | Макс. изменение курса за шаг |
| σ_wV | ΔV_max / 3 | СКО шума скорости (правило 3σ) |
| σ_wψ | Δψ_max / 3 | СКО шума курса (правило 3σ) |

Матрица шума процесса: `Q = diag(0, 0, σ²_wV, σ²_wψ)` — координаты являются интегральными величинами.

### Параметры НПА (раздел 2.1)

Физические параметры аппарата и среды задаются в `auv_params.py`.
На их основе автоматически вычисляются ковариационные матрицы Q и P0:

| Параметр               | Обозначение | Значение        |
|------------------------|-------------|-----------------|
| Масса                  | m           | 50 кг           |
| Максимальная скорость  | V_max       | 2.0 м/с         |
| Время разгона          | τ           | 3.0 с           |
| Макс. угл. скорость    | ψ̇_max      | 5 °/с           |
| Шаг дискретизации      | Δt          | 0.1 с           |
| Плотность воды         | ρ           | 1025 кг/м³      |
| Температура воды       | T           | 20 °C           |
| Скорость звука         | c           | 1480 м/с        |

**Вычисляемые величины (правило 3σ):**

```
ΔV_max  = (V_max / τ) * Δt = (2.0 / 3.0) * 0.1 ≈ 0.0667 м/с
Δψ_max  = ψ̇_max * Δt       = deg2rad(5) * 0.1  ≈ 0.00873 рад
σ_wV    = ΔV_max / 3        ≈ 0.0222 м/с
σ_wψ    = Δψ_max / 3        ≈ 0.00291 рад

Q  = diag(0, 0, σ²_wV, σ²_wψ)
P0 = diag(100, 100, 0.25, deg2rad(0.5)²)

σ_DVL = sqrt(σ²_base + σ²_temp) = sqrt(0.02² + 0.01²) ≈ 0.0224 м/с
      → округлено до 0.03 м/с (наихудший случай)
```

## Принципы архитектуры

- **S** — Single Responsibility: каждый модуль отвечает за одну задачу
- **O** — Open/Closed: новые фильтры/датчики добавляются без изменения существующего кода
- **L** — Liskov Substitution: любой фильтр заменяем через абстрактный интерфейс
- **I** — Interface Segregation: минимальные интерфейсы для каждой абстракции
- **D** — Dependency Inversion: модули зависят от абстракций, не от реализаций

## Экспериментальные сценарии (глава 4)

### Сценарий 1 — Круговая траектория (пункт 4.2)
Базовое сравнение EKF, Adaptive EKF и Dead Reckoning.
Показывает, что оба фильтра значительно точнее наивного счисления пути.

### Сценарий 2 — Прямолинейная траектория (пункт 4.3)
Проверка на простейшем маршруте.
Подтверждает работоспособность на другом типе движения.

### Сценарий 3 — Восьмёрка (пункт 4.3)
Сложный маршрут с частой сменой курса.
Проверяет устойчивость при интенсивном маневрировании.

### Сценарий 4 — Сбои датчиков (пункт 4.4)
DVL отключается на 30 с (100–130 с), компас — на 10 с (150–160 с).
Демонстрирует преимущество адаптивного EKF при нестационарных условиях.

### Dead Reckoning как baseline (пункт 4.5)
Наивное счисление пути без коррекции.
Ошибка растёт неограниченно, обосновывая необходимость фильтрации.

### Интерпретация результатов
- Если адаптивный EKF немного хуже классического при стационарном шуме —
  это ожидаемо: адаптация добавляет шум в оценку R без необходимости.
- При сбоях датчиков адаптивный EKF должен показать преимущество.
- Результаты воспроизводимы при фиксированном `seed=42`.
